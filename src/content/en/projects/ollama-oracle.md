# Problem
I needed to make internal knowledge available in conversational form, but without giving up privacy and cost predictability, avoiding dependence on external APIs. Content was scattered across text and structured records, which required a consistent way to turn it into retrievable context. In addition, it was important to reduce "made up" answers and be able to justify the response with references to the material used. Finally, I needed a responsive user experience, with incremental response, without freezing the interface on longer queries.

# Solution
I implemented an ingestion pipeline that scans documents, normalizes common formats, and splits content into overlapping chunks to improve retrieval. These chunks are vectorized by a local embedding model and persisted in a vector database, enabling similarity search with score.

At chat runtime, I apply distance filters and context limits to control noise, plus deduplicate repeated chunks before assembling the prompt. The response is generated by a local model and delivered via streaming events, with automatic cancellation when the client disconnects to avoid unnecessary work.

## Highlights
- **Ingestion** pipeline with normalization and overlapping chunking
- **Local embeddings** + vector database for similarity search with score
- Quality controls: **filters**, context limits, and **deduplication** in retrieval
- **Streaming** response and cancellation on disconnect to avoid waste

# Impact
In practice, the system began answering based on evidence from indexed content, reducing hallucinations and making output more auditable through sources. The separation between ingestion, retrieval, and generation kept quality and performance adjustments focused on parameters, without rewriting the main flow. Streaming improved perceived latency and the abort routine on disconnects avoided CPU waste and queues on the local model. The result was a conversational base that scales by document volume with repeatable reindexing and simple maintenance, since each critical integration stays isolated and testable by behavior.
